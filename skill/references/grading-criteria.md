# Grading Criteria by Domain

## Code / Backend

| Criterion | Weight | PASS | FAIL |
|---|---|---|---|
| **Functionality** | High | All DoD criteria met, features work as specified | Any DoD criterion not met, stubs present |
| **Completude** | High | No TODOs, no stubs, no placeholders | Any stub or unfinished code |
| **Robustez** | Medium | Edge cases handled, errors caught, invalid input rejected | Crashes on bad input, unhandled errors |
| **Qualidade** | Medium | Clean code, follows project conventions, no hacks | Copy-paste, magic numbers, inconsistent style |
| **Tests** | High | New tests for new code, all tests pass | Missing tests, broken tests |
| **Types** | Medium | TypeScript compiles clean (`tsc --noEmit` = 0 errors) | Type errors present |

## Frontend / UI

| Criterion | Weight | PASS | FAIL |
|---|---|---|---|
| **Design quality** | High | Coherent visual identity, intentional choices | Generic template, AI slop patterns |
| **Originality** | Medium | Custom decisions, distinct character | Unmodified defaults, purple gradients on white cards |
| **Craft** | Medium | Good typography, spacing, contrast, responsive | Broken layout, inconsistent spacing, poor contrast |
| **Functionality** | High | Users can complete tasks, clear affordances | Confusing UX, broken interactions, dead buttons |

## Research / Content

| Criterion | Weight | PASS | FAIL |
|---|---|---|---|
| **Profundidade** | High | Substantive analysis, multiple angles | Surface-level, obvious conclusions only |
| **Accuracy** | High | Facts verifiable, sources cited | Unsourced claims, hallucinated data |
| **Completude** | Medium | Covers all requested angles | Missing sections, incomplete analysis |
| **Actionability** | Medium | Concrete recommendations, next steps | Vague suggestions, no clear takeaways |
